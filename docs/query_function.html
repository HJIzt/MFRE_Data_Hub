Creating fundamentals_querier()

I want a function that gets us to the same starting point as where Power Query leaves us - with the cashflow, balance sheet, and income statement of whatever desired company tickers we input.

It needs to do several things:

  1. Take a list of stock tickers
  2. For each ticker, query cashflow, balance sheet, and income statement
  3. Join each company’s reports together
  4. Concatenate these (with years matched) together for all companies queried
  5. Return this dataframe
import datetime
import numpy as np


def fundamentals_querier(tickers):
    """_summary_

    Args:
        tickers (list of strings): a list of the string tickers of desired companies
    """
    # create the dictionary I'm associating the tickers and their fundamentals data in
    fundamentals_dict = {}
    
    # execute a set of operations for every individual `ticker` in the list `tickers` 
    for ticker in tickers:
        # query the three dataframes
        cash_flow = si.get_cash_flow(ticker)
        balance_sheet = si.get_balance_sheet(ticker)
        income_statement = si.get_income_statement(ticker)
        
        # concatenate the above three
        ticker_fundamentals = pd.concat([cash_flow, balance_sheet, income_statement])
        
        # change the long date columns to overall year columns
        ticker_fundamentals.columns = pd.to_datetime(ticker_fundamentals.columns.values).year
        
        # assign name of the company to the data
        ticker_fundamentals["ticker"] = ticker
        
        # add the ticker and fundamentals to the dictionary
        fundamentals_dict[ticker] = ticker_fundamentals
        
    
    # create fundamentals_df, which is all the data from the different tickers, concatenated into one
    fundamentals_df = pd.concat(fundamentals_dict.values())
    
    # return the fundamentals_df. not print(), since we want to do more with it later!
    return fundamentals_df

If I wanted, I could just run the central block of code multiple times for every stock ticker to analyze, or keep it as a non-function For loop. But writing this up as a function lets us very easily do it multiple times, and I could import this function to other Python work and easily apply it there. So, building it as a function makes some sense.

So can we plot our data?

We can’t plot anything useful, yet.

In Pandas, you can very easily do operations on entire columns at once. We can add, subtract, multiply, and divide columns just like numbers in basic Python. This is important for our calculated fields.

For example;

df1["new_column"] = df1["column_a"]*df1["column_b"]

This creates "new_column" in df1, where each row is equal to that row’s "column_a" values multiplied by the row’s "column_b" value.

For this to be useful, the columns must all be different variables, and their rows must be meaningfully related. Since we want to compute the financial variables for a company in a given year, we want each row to correspond to a given company in one year.

Right now, big_tech has its variable names as a row variable, while the years and companies are columns. We need to do some work first.

Tidy vs Untidy data

From R for Data Science:

There are three interrelated rules which make a dataset tidy:

1. Each variable must have its own column.
2. Each observation must have its own row.
3. Each value must have its own cell.

At the end of the PowerQuery process, the data was tidy.

Why is Tidy Data good?

In tidy data there is a meaningful relation to values in the same column (they’re for the same variable) and values in the same row (they belong to the same observation).

This means you can input a column to plot, or as a variable in a model, or apply an aggregation like finding the mean or variance.

It also allows you to derive new values from your data by multiplying, dividing, adding, or subtracting column values from each other, like our calculated items in Excel.

Problems with big_tech

There are two main ones, from a tidy data perspective.

  1. The year variable is spread out across multiple columns.

  2. The different financial variables in the Breakdown column should be each their own column.

Generating our calculated fields and plotting them will be a hassle at present. We’ll handle these problems in order.

Consolidating years into one variable

This table is in wide format - we’ve got multiple years of the same variable in each row.

The Pandas melt function lets us specify fixed identifier variables. These are used to state things like entities (eg. which company is being observed), variable names, or other features that link the identity of observations. Then, the other columns get made into a new variable, which we specify the name of with the var_name argument.

So in our case, we list "ticker" and "Breakdown" as our id_vars, and the new var_name as "Year". Melt keeps the "ticker" and "Breakdown" columns, but each pair gets their year values turned into a separate row, with the corresponding variable value as the new "value" column.

big_tech_melt = pd.melt(big_tech.reset_index(), id_vars = ["ticker", "Breakdown"], var_name = "Year")

As you can see, every column corresponds to a different variable. The first two are "ticker" and "Year", which identify the observations. The remainder are financial fundamentals data. We’re ready to do our calculated fields next.

This table is equivalent to the second pivot table we made in Excel, where the years are in the columns. There isn’t an easy way to include a slicer for years in Pandas, though we can easily filter the dataset by year. We’ll leave it as the whole time range for now.

What did we just do?

It’s worth outlining what we’ve done in this series of dataframe transformations.

Our first dataframe, big_tech, was very simple. One column for what "Breakdown" is being covered, one for which company’s "ticker", and each column after that is a year. The rows have values for that breakdown, company, and year. This wasn’t very useful. We want to compare each company’s variables across years, and it’s hard to plot that from this format. As well, matching multiple variables in the same year is very hard, so we can’t make our calculated fields.

The melt() Pandas function takes you from “wide” format (where one variable’s information is in multiple columns) and to “long” format (by breaking the rows apart and stacking them up). So, we melted big_tech to big_tech_melt. This turned each row into a ticker/year/breakdown combo, which is an improvement. But we still need to get the columns to be the different variables for each company, matched by year. Each row has to be for one company for one year. big_tech_melt cannot offer this yet.

By using a pivot table, we got there. table_2_equiv is our version of the second pivot table (with years in the columns) in the Excel section. The dataframe is a grid. Every column is a different variable, and every row corresponds to a single company in a single year. Now that it’s Tidy, it’s easy to do column operations, plotting, modelling, or whatever else we feel like.
